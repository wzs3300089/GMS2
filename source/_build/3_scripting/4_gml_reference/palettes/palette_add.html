palette_add


Description

This function can be used to set the alpha value for any of the 64 palette indices of a given palette resource. You supply the unique palette ID to set the indices of as well as the index in the palette to use (from 0 to 63), and finally the alpha value (from 0 to 255). If you need to set the colour value use palette_set_colour, or if you need to set the colour and alpha value use palette_set). Note that a new palette will by default have a colour value of $00ffffff (white, with 0 alpha) for each palette index.

IMPORTANT! When you create a new resource at runtime, you need to be sure to remove it again when you no longer need it or before you call a game_restart or room_restart, using palette_delete, otherwise you run the risk of getting a memory leak. This happens when you create a resource and then remove or over-write the reference to this resource, meaning that it is taking up memory but is no longer available to GameMaker Studio 2 to use.


Syntax:

palette_add();


Returns:

Real (ID value for the resource)


Example:

global.pLight= palette_add();
var _colArray;
for (var i = 0; i < 64; i++;)
    {
    var _col = make_colour_hsv(random(255), 255, 255);
    _colArray[i] = _col | $ff000000;
    }
palette_set(global.pRandom, _colArray);

The above code creates a new palette resource and stores its ID in a global variable. It then creates a 64 index array of random colour values and adds an alpha component before passing this array backk to the palette.
