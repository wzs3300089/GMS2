draw_tile_ext

Description

With this function you can draw any given tile from a tile set - compete with rotations, flips and mirrors - setting the frame (if animated) and the position within the room, as well as the palette that should be used for drawing. The tile set ID value is the tile set index as set in the IDE and the tile data is the data "blob" that contains all the information about the tile and the transformations, palette blend, and animation index that have been applied. You can use the function tilemap_get to get the tile data which can then be manipulated using the Tile Functions, or you can construct it yourself using the following tile constants along with the index value of the tile to create:

Constant Description
tile_rotate Used to set/get the rotate bit of a tile data blob.
tile_mirror Used to set/get the mirror bit of a tile data blob.
tile_flip Used to set/get the flip bit of a tile data blob.
tile_index_mask A special constant that is for "and"-ing with the tile data blob to extract the tile index.


You would bitwise "or" these constants with the tile index value to get the final tiledata:

var tiledata = 8 | tile_rotate | tile_flip;


The above code simply creates a local variable to store the tiledata for the tile with the index of 8 on the tile set, and it is rotated and flipped. If you want to change the palette blending index (the index of the 64 value palette array for blending) then you will need to use the function tile_set_palette_entry on the tile data as shown here:

var tiledata = 8 | tile_rotate | tile_flip;
tiledata = tile_set_palette_entry(tiledata, 35);


The above code sets the index of the tile, the rotate and flip values and sets the palette index to 35. This tiledata blob can then be used to draw the tile at that position from the tile set that you specify and blended with the colour indexed in the palette array at 35.

Note that this function is not linked to any tilemap layer and the tile will be drawn at the layer depth of the instance doing the drawing, at the x/y position that you specify in the room, using the palette that you specify, and the frame that you specify. If the indexed tile is part of an animation set, then you can set the frame number in this function as well (from 0 to the number of frames - 1) and the indexed tile will simply be the indicator for which animated tile set you are using while it will be the given frame that is drawn. If the tile is not part of an animated set then frame should be set to 0.


Syntax:

draw_tile_ext(tileset, tiledata, frame, x, y, palette_id)


Arguments:

Argument Description
tileset The unique ID of the tile set to use
tiledata The tile data to use
frame The frame number for the animated tile to use (default 0 for non-animated tiles)
x The x position within the room to draw at
y The y position within the room to draw at
palette_id The unique ID value of the palette resource to use


Returns:

Real


Example:

var lay_id = layer_get_id("Tiles_sky");
var map_id = layer_tilemap_get_id(lay_id);
var palette_id = tilemap_get_palette(map_id)
var data = tilemap_get(map_id, 0, 0);
if ++global.palette_index >63 global.palette_index = 0;
data = tile_set_palette_entry(data, global.palette_index);
draw_tile_ext(set_Wall, data, 0, x, y, palette_id);

The above code will get the data on a tile from the given tilemap along with the unique palette ID value for the palette assigned to the layer. The palette index value stored in the global variable is then incremented and used to change the value stored in the tile data before being used to draw the tile.
